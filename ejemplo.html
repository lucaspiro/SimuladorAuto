📖 Ejemplo de Uso

Guía básica para programar tu robot en el simulador de esquiva de obstáculos

1. ¿Cómo funciona?

Tu código se ejecuta en cada frame (60 veces por segundo). Dentro tenés acceso a los sensores, las funciones de movimiento y las variables de estado. Pensá que tu código reemplaza al void loop() de Arduino.

💡 Tip: Usá millis() en vez de delay(). El simulador no soporta delay porque bloquearía el navegador.

2. Referencia de APIs

🚗 Funciones de movimiento

FunciónDescripción
avanzar()Mueve el robot hacia adelante
retroceder()Mueve el robot hacia atrás
girarDer()Gira sobre su eje a la derecha
girarIzq()Gira sobre su eje a la izquierda
parar()Detiene ambos motores

📡 Sensores

VariableDescripción
sensor_frontalDistancia al obstáculo más cercano adelante (0 a 250)
sensor_izquierdoDistancia al obstáculo a la izquierda (0 a 250)
sensor_derechoDistancia al obstáculo a la derecha (0 a 250)

🏷️ Estados

ConstanteUso
AVANZAREstado por defecto, el robot avanza
GIRAR_DCorrección: girando a la derecha
GIRAR_ICorrección: girando a la izquierda
RETROCEDERMarcha atrás por un tiempo definido
GIRAR_ESTRAT_IGiro estratégico izquierda (con temporizador)
GIRAR_ESTRAT_DGiro estratégico derecha (con temporizador)

🔧 Variables de control

VariableDescripción
estadoContiene el estado actual (ej: estado = AVANZAR)
tControlGuarda el momento en que se entró al estado
tDuracionCuántos milisegundos dura el estado actual

⏱️ Utilidades

FunciónDescripción
millis()Devuelve los milisegundos transcurridos (como en Arduino)
random(min, max)Devuelve un número entero aleatorio entre min y max-1

3. Ejemplo simple: Avanzar y esquivar

Este es el código más básico para que el robot avance y, cuando detecte algo de frente, retroceda y gire:

// Ejemplo básico: avanzar y esquivar

switch (estado) {

case AVANZAR:
  avanzar();
  // Si hay algo cerca de frente, retroceder
  if (sensor_frontal < 60) {
    estado = RETROCEDER;
    tControl = millis();
    tDuracion = 500;
  }
  break;

case RETROCEDER:
  retroceder();
  if (millis() - tControl >= tDuracion) {
    // Giro random para no repetir siempre lo mismo
    if (random(0, 2) === 0) {
      estado = GIRAR_ESTRAT_I;
    } else {
      estado = GIRAR_ESTRAT_D;
    }
    tDuracion = 300;
    tControl = millis();
  }
  break;

case GIRAR_ESTRAT_I:
  girarIzq();
  if (millis() - tControl >= tDuracion) {
    estado = AVANZAR;
  }
  break;

case GIRAR_ESTRAT_D:
  girarDer();
  if (millis() - tControl >= tDuracion) {
    estado = AVANZAR;
  }
  break;
}
⚠️ Importante: Este ejemplo es intencionalmente simple y básico. No usa los sensores laterales ni tiene corrección de centrado. ¡Tu desafío es mejorarlo para que esquive mejor!

4. Patrón clave: millis() en vez de delay()

En Arduino real usarías delay(1000) para esperar 1 segundo. Acá no se puede porque bloquea todo. En su lugar, usamos este patrón:

// ❌ MAL - No funciona en el simulador:
retroceder();
delay(1000);  // Bloquea todo

// ✅ BIEN - Patrón con millis():
case RETROCEDER:
  retroceder();
  if (millis() - tControl >= 1000) {
    // Ya pasó 1 segundo, cambiar estado
    estado = AVANZAR;
  }
  break;
💡 Tip: Guardá tControl = millis() cuando entrás al estado, y después chequeá si ya pasó el tiempo comparando millis() - tControl >= tDuracion.
